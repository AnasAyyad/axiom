# ADR-NNNN: Decision title

- **Status:** Proposed
- **Date:** YYYY-MM-DD
- **Scope:** Release/phase and affected subsystem

## Context

State the requirement, uncertainty, and constraints that require a durable
decision. Link stable requirement IDs and threat items.

## Decision

State the normative design, ownership, failure behavior, versioning, and safety
boundaries. An ADR cannot weaken the specification.

## Consequences

Record benefits, costs, operational effects, compatibility, migration, and
remaining limitations.

## Rejected alternatives

- Alternative: concrete reason it was rejected.

## Validation

List exact tests, scans, benchmarks, drills, evidence locations, and rollback or
forward-compatibility checks.

## Revisit when

List objective triggers. A changed accepted decision requires a superseding ADR.
