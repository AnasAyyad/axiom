# V1A requirements traceability

The normative V1A requirement catalogue and verification matrix is
[requirements/traceability.md](requirements/traceability.md).
Its clause-level source audit is
[requirements/source-coverage.md](requirements/source-coverage.md).

This path is retained because the V1A implementation plan names it literally.
Requirement IDs, owners, targets, planned verification, evidence locations, and
statuses must be changed only in the canonical document. A requirement is not
complete merely because it appears in either document.
